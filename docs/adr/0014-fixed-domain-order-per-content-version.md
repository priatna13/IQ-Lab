# Domain order is fixed per Content Version and pinned on the Attempt

Order effects exist, but MVP prioritizes a single comparable instrument over randomized sequences. Each Content Version defines both the Item Bank and the fixed Domain sequence. That sequence is pinned on Attempt creation with the Content Version. Participants cannot reorder Domains; changing order is an editorial new Content Version, not a live tweak to open Attempts.

**Consequences:** Runner advances Domains by versioned sequence; analytics can treat position fatigue as stable within a version; randomization deferred post-MVP.
