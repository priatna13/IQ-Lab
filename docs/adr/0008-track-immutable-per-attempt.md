# Track is fixed when an Attempt is created

Track changes framing, action-plan templates, and LLM narration, not the Item set. Allowing mid-Attempt or post-hoc track switches blurs what a Report means and invites “try both framings” without a clear Primary. MVP locks Track at Attempt creation. Changing mind requires abandoning the Open Attempt (no retake cooldown) and starting a new Open Attempt when policy allows. Post-complete reframe is deferred.

**Consequences:** Onboarding must confirm track before first Domain Session; analytics can attribute completion by track cleanly.
